# AI Email Assistant

## Описание проекта

Данный проект представляет собой систему автоматической обработки входящих email-сообщений с использованием AI-модели.

Система:

1. Получает письмо от пользователя.
2. Анализирует содержание письма.
3. Определяет:
   - суть вопроса,
   - эмоциональный окрас сообщения.
4. Генерирует ответ с указанием уровня уверенности модели.
5. Сохраняет все данные в базу данных и отображает их в веб-таблице.

Если уровень уверенности модели ниже заданного порога `n`, в процесс обработки подключается оператор.

---

## Логика работы системы

### 1. Получение письма

Пользователь отправляет вопрос на почту, подключенную к системе.

### 2. Анализ сообщения

AI-модель:
- извлекает основную суть запроса,
- определяет эмоциональный тон (позитивный, негативный, нейтральный и т.д.),
- генерирует ответ,
- вычисляет уровень уверенности (confidence score).

### 3. Проверка уверенности

- Если `confidence >= n`  
  Ответ автоматически отправляется пользователю.

- Если `confidence < n`  
  Ответ не отправляется.  
  Подключается оператор.

### 4. Работа оператора

Оператор:

- Просматривает предложенный AI ответ.
- Принимает решение:
  - Отправить ответ без изменений
  - Отредактировать или написать новый ответ

После этого:
- Финальный ответ записывается в базу данных.
- Данные обновляются в веб-таблице.

### 5. Отправка финального ответа

После определения точного ответа отправляется пользователю на email.

## Бизнес-логика порога уверенности

Порог `n` задается в конфигурации системы.

Пример:

```python
CONFIDENCE_THRESHOLD = 0.85
```

Это позволяет:

- Минимизировать риск некорректных автоответов
- Снизить нагрузку на операторов
- Контролировать качество коммуникации

---

## Структура веб-таблицы

Система использует две связанные таблицы.

### 1. Основная таблица обращений

Поля таблицы:

- Дата  
- ФИО  
- Объект  
- Телефон  
- Email  
- Заводские номера  
- Тип приборов  
- Эмоциональный окрас  
- Суть вопроса  
- UID вопроса (уникальный ключ)

### 2. Таблица сообщений (связь по UID вопроса)

Поля таблицы:

- UID сообщения  
- Текст сообщения  
- Сообщение от оператора  
- Сообщение-ответ-1 (от ИИ)  
- Сообщение-ответ-2 (от оператора)  
- Параметр уверенности ответа ИИ  

Связь между таблицами осуществляется по полю UID вопроса.
